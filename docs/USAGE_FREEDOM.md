# Usage Freedom

Calendar Add is shared under the [WTFPL](../LICENSE).

Plain meaning:

- Use this project for anything.
- Copy it.
- Fork it.
- Modify it.
- Publish your own version.
- Sell it if you want.
- Redistribute it.
- Reuse parts of it in another project.
- You do not need to ask permission.

The intent is simple: this project is shared freely. Do what you want with it.

## Contributions

Contributions accepted into this repository are also shared under the WTFPL.

The contributor guidelines only explain how to get changes merged into this repository. They do not limit your right to fork, modify, publish, or use the project independently.

## Project Spaces

The code of conduct applies to participation in this repository's official project spaces, such as issues, pull requests, and maintainer discussions.

It does not add extra restrictions to the WTFPL license and does not limit private use, forks, redistribution, or derivative works.

## Security Reports

The security policy asks people to report vulnerabilities privately first. That is a responsible-disclosure request, not a license restriction.

## Third-Party Material

The WTFPL applies to Calendar Add code and documentation owned by this project.

Third-party dependencies, Android tooling, LiteRT-LM libraries, and downloaded AI model files may have their own licenses or terms. This project does not add restrictions on top of them, but it also cannot remove restrictions from material owned by others.
