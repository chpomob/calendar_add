# Security Policy

## Reporting a Vulnerability

We take the security of Calendar Add AI seriously. If you discover a security
vulnerability, please report it responsibly.

### How to Report

1. **Do NOT** create a public GitHub issue
2. Email: chpomob@gmail.com
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Your contact information

### Response Time

- Initial acknowledgment: Within 48 hours
- Update on remediation: Within 72 hours
- Resolution: Within 14 days (critical)

### What We Do NOT Accept

- Generic vulnerability reports
- Requests for malicious tools
- Exploit kits or malware

### Scope

We accept reports for:
- Code vulnerabilities
- Data privacy issues
- Authentication bypasses
- Third-party integrations

## Security Best Practices

### For Users

1. **Keep the app updated**
2. **Use a secure device** (lock screen, security software)
3. **Don't jailbreak/root** your device
4. **Only install from trusted sources**

### For Contributors

1. **Follow security guidelines**
2. **Review third-party dependencies**
3. **Test for vulnerabilities**
4. **Use secure coding practices**

## Vulnerability Disclosure Policy

We follow a coordinated disclosure process:

1. Report vulnerability privately
2. We investigate and assess
3. We provide remediation timeline
4. We credit if you wish
5. We publish announcement (after fix)

## Security Measures

Our app implements:

- ✅ ProGuard obfuscation
- ✅ Encrypted storage
- ✅ No data exfiltration
- ✅ Regular dependency updates
- ✅ Security-focused testing

## Third-Party Security

We use:
- TensorFlow Lite (official)
- AndroidX libraries
- Google Play Billing

See their security policies for details.
